== Vision Produit

Un plugin Gradle N4 qui **assiste la code review de PRs via IA** (gpt-oss:120b-cloud),
produit un rapport structuré, vérifie les conventions du workspace et les quality gates.

'''
== User Stories

=== US-RVW-01 — En tant que reviewer, je veux une pre-review IA qui m'économise du temps

*Priorité* : P1
*EPIC* : RVW-2

.Critères d'acceptation
- `reviewPr` prend une PR GitHub (numéro ou URL) et analyse le diff
- L'IA (gpt-oss:120b-cloud) lit le diff et produit un rapport structuré
- Le rapport couvre : anomalies, suggestions, secrets, patterns, DAG
- Le rapport est posté en commentaire sur la PR

=== US-RVW-02 — En tant que reviewer, je veux un score de qualité automatique

*Priorité* : P1
*EPIC* : RVW-3

.Critères d'acceptation
- `reviewScore` attribue un score 1-5 basé sur des critères objectifs
- Critères : secrets détectés, conventional commits, patterns respectés, tests présents, complexité
- Le score est ajouté au commentaire de la PR

=== US-RVW-03 — En tant que lead dev, je veux des quality gates bloquantes

*Priorité* : P2
*EPIC* : RVW-3

.Critères d'acceptation
- `reviewGate` vérifie les règles absolues du workspace (AGENTS.adoc)
- Gate 1 : aucun secret en dur
- Gate 2 : conventional commits
- Gate 3 : pas d'import N3→N2
- Une gate non passée = PR bloquée (label "needs-fix")

=== US-RVW-04 — En tant que chef de projet, je veux un rapport de review consolidé

*Priorité* : P3
*EPIC* : RVW-4

.Critères d'acceptation
- `reviewReport` génère un rapport AsciiDoc par PR
- Métriques : score, anomalies trouvées, suggestions acceptées/refusées
- Historique des reviews conservé
